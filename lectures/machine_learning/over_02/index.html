    Overfitting II¬∂

    Algunas de las t√©cnicas que podemos utilizar para reducir el overfitting, son:

    • Recolectar m√°s datos.
    • Introducir una penalizaci√≥n a la complejidad con alguna t√©cnica de regularizaci√≥n.
    • Utilizar modelos ensamblados.
    • Utilizar validaci√≥n cruzada.
    • Optimizar los par√°metros del modelo con grid search.
    • Reducir la dimensi√≥n de los datos.
    • Aplicar t√©cnicas de selecci√≥n de atributos.

    Veremos ejemplos de algunos m√©todos para reducir el sobreajuste (overfitting).

    Validaci√≥n cruzada¬∂

    La validaci√≥n cruzada se inicia mediante el fraccionamiento de un conjunto de datos en un n√∫mero $k$ de particiones (generalmente entre 5 y 10) llamadas pliegues.

    La validaci√≥n cruzada luego itera entre los datos de evaluaci√≥n y entrenamiento $k$ veces, de un modo particular. En cada iteraci√≥n de la validaci√≥n cruzada, un pliegue diferente se elige como los datos de evaluaci√≥n. En esta iteraci√≥n, los otros pliegues $k-1$ se combinan para formar los datos de entrenamiento. Por lo tanto, en cada iteraci√≥n tenemos $(k-1) / k$ de los datos utilizados para el entrenamiento y $1 / k$ utilizado para la evaluaci√≥n.

    Cada iteraci√≥n produce un modelo, y por lo tanto una estimaci√≥n del rendimiento de la generalizaci√≥n, por ejemplo, una estimaci√≥n de la precisi√≥n. Una vez finalizada la validaci√≥n cruzada, todos los ejemplos se han utilizado s√≥lo una vez para evaluar pero $k -1$ veces para entrenar. En este punto tenemos estimaciones de rendimiento de todos los pliegues y podemos calcular la media y la desviaci√≥n est√°ndar de la precisi√≥n del modelo.

    Optimizaci√≥n de par√°metros con Grid Search¬∂

    La mayor√≠a de los modelos de Machine Learning cuentan con varios par√°metros para ajustar su comportamiento, por lo tanto, otra alternativa que tenemos para reducir el Sobreajuste es optimizar estos par√°metros por medio de un proceso conocido como grid search e intentar encontrar la combinaci√≥n ideal que nos proporcione mayor precisi√≥n.

    El enfoque que utiliza grid search es bastante simple, se trata de una b√∫squeda exhaustiva por el paradigma de fuerza bruta en el que se especifica una lista de valores para diferentes par√°metros, y la computadora eval√∫a el rendimiento del modelo para cada combinaci√≥n de √©stos par√°metros para obtener el conjunto √≥ptimo que nos brinda el mayor rendimiento.

    Reducci√≥n de dimensionalidad¬∂

    La reducci√≥n de dimensiones es frecuentemente usada como una etapa de preproceso en el entrenamiento de sistemas, y consiste en escoger un subconjunto de variables, de tal manera, que el espacio de caracter√≠sticas quede √≥ptimamente reducido de acuerdo a un criterio de evaluaci√≥n, cuyo fin es distinguir el subconjunto que representa mejor el espacio inicial de entrenamiento.

    Como cada caracter√≠stica que se incluye en el an√°lisis, puede incrementar el costo y el tiempo de proceso de los sistemas, hay una fuerte motivaci√≥n para dise√±ar e implementar sistemas con peque√±os conjuntos de caracter√≠sticas. Sin dejar de lado, que al mismo tiempo, hay una opuesta necesidad de incluir un conjunto suficiente de caracter√≠sticas para lograr un alto rendimiento.

    La reducci√≥n de dimensionalidad se puede separar en dos tipos: Extracci√≥n de atributos y Selecci√≥n de aributos.

    Extracci√≥n de atributos¬∂

    La extracci√≥n de atributos comienza a partir de un conjunto inicial de datos medidos y crea valores derivados (caracter√≠sticas) destinados a ser informativos y no redundantes, lo que facilita los pasos de aprendizaje y generalizaci√≥n posteriores, y en algunos casos conduce a a mejores interpretaciones humanas.

    Cuando los datos de entrada a un algoritmo son demasiado grandes para ser procesados y se sospecha que son redundantes (por ejemplo, la misma medici√≥n en pies y metros, o la repetitividad de las im√°genes presentadas como p√≠xeles), entonces se puede transformar en un conjunto reducido de caracter√≠sticas (tambi√©n denominado un vector de caracter√≠sticas).

    Estos algoritmos fueron analizados con profundidad en la secci√≥n de An√°lisis no supervisados - Reducci√≥n de la dimensionalidad.

    Selecci√≥n de atributos¬∂

    Proceso por el cual seleccionamos un subconjunto de atributos (representados por cada una de las columnas en un datasetde forma tabular) que son m√°s relevantes para la construcci√≥n del modelo predictivo sobre el que estamos trabajando.

    El objetivo de la selecci√≥n de atributos es :

    • mejorar la capacidad predictiva de nuestro modelo,
    • proporcionando modelos predictivos m√°s r√°pidos y eficientes,
    • proporcionar una mejor comprensi√≥n del proceso subyacente que gener√≥ los datos.

    Los m√©todos de selecci√≥n de atributos se pueden utilizar para identificar y eliminar los atributos innecesarios, irrelevantes y redundantes que no contribuyen a la exactitud del modelo predictivo o incluso puedan disminuir su precisi√≥n.

    Algoritmos para selecci√≥n de atributos

    Podemos encontrar dos clases generales de algoritmos de selecci√≥n de atributos: los m√©todos de filtrado, y los m√©todos empaquetados.

    • M√©todos de filtrado: Estos m√©todos aplican una medida estad√≠stica para asignar una puntuaci√≥n a cada atributo. Los atributos luego son clasificados de acuerdo a su puntuaci√≥n y son, o bien seleccionados para su conservaci√≥n o eliminados del conjunto de datos. Los m√©todos de filtrado son a menudo univariantes y consideran a cada atributo en forma independiente, o con respecto a la variable dependiente.

      • Ejemplos : prueba de Chi cuadrado, prueba F de Fisher, ratio de ganancia de informaci√≥n y los coeficientes de correlaci√≥n.

    • M√©todos empaquetados: Estos m√©todos consideran la selecci√≥n de un conjunto de atributos como un problema de b√∫squeda, en donde las diferentes combinaciones son evaluadas y comparadas. Para hacer estas evaluaciones se utiliza un modelo predictivo y luego se asigna una puntuaci√≥n a cada combinaci√≥n basada en la precisi√≥n del modelo.

def classification_metrics(df: pd.DataFrame) -> pd.DataFrame:
    """

    :return: dataframe con las m√©tricas especificadas
    """
    y_true = df['y']
    y_pred = df['yhat']

    accuracy = round(accuracy_score(y_true, y_pred), 4)
    recall = round(recall_score(y_true, y_pred, average='macro'), 4)
    precision = round(precision_score(y_true, y_pred, average='macro'), 4)
    fscore = round(f1_score(y_true, y_pred, average='macro'), 4)

    df_result = pd.DataFrame({'accuracy': [accuracy],
                              'recall': [recall],
                              'precision': [precision],
                              'fscore': [fscore]})

    return df_result
