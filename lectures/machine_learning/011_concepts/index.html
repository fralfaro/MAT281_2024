    Representaci√≥n de Datos¬∂

    Una tabla es una cuadr√≠cula de datos organizada en filas y columnas. Las filas corresponden a elementos individuales del conjunto, mientras que las columnas representan atributos o caracter√≠sticas de estos elementos.

    Por ejemplo, el conjunto de datos Iris, estudiado por Ronald Fisher en 1936, utiliza esta estructura para mostrar las caracter√≠sticas de diferentes especies de flores.

    Descripci√≥n de los columnas

    • sepal_length: Longitud del s√©palo en cent√≠metros.
    • sepal_width: Ancho del s√©palo en cent√≠metros.
    • petal_length: Longitud del p√©talo en cent√≠metros.
    • petal_width: Ancho del p√©talo en cent√≠metros.
    • species: Especie de la flor (setosa, versicolor, virginica).

    Conjunto de Entrenamiento y Prueba¬∂

    El conjunto de entrenamiento (train set) y el conjunto de prueba (test set) son dos subconjuntos de datos utilizados en aprendizaje autom√°tico.

    El primero se usa para entrenar el modelo, mientras que el segundo se reserva para evaluar su rendimiento y capacidad de generalizaci√≥n.

    • El conjunto de entrenamiento es un subconjunto de datos utilizado para ajustar los par√°metros del modelo y aprender la relaci√≥n entre las variables de entrada y la salida. En esencia, sirve para entrenar al modelo.

    • El conjunto de prueba es otro subconjunto, separado del proceso de entrenamiento, empleado para evaluar el rendimiento del modelo. Permite medir su capacidad de generalizaci√≥n al enfrentarlo a datos nuevos y no vistos previamente.

    La funci√≥n train_test_split de Scikit-learn es una herramienta esencial para dividir un conjunto de datos en subconjuntos de entrenamiento y prueba. Permite especificar la proporci√≥n de datos para cada conjunto, siendo 75% para entrenamiento y 25% para prueba de forma predeterminada. Adem√°s, toma como entrada tanto los datos como las etiquetas correspondientes.

    Reglas de separaci√≥n

    En general, los datos se dividen en conjuntos de entrenamiento y prueba. El tama√±o de cada conjunto puede variar seg√∫n el n√∫mero de filas y la complejidad del modelo. Por lo general, el conjunto de entrenamiento ocupa entre el 70% y el 80%, mientras que el de prueba ocupa entre el 20% y el 30%.

    Error de Estimaci√≥n¬∂

    El error de estimaci√≥n es la diferencia entre los valores reales y los valores predichos por un modelo. En aprendizaje autom√°tico, se refiere a la discrepancia entre la variable objetivo (dependiente) y la predicci√≥n del modelo. El objetivo principal es minimizar este error para mejorar la precisi√≥n en las predicciones.

    El c√°lculo del error de estimaci√≥n depende del tipo de datos en la variable objetivo:

    • Num√©ricos: Se utilizan m√©tricas como el Error Cuadr√°tico Medio (MSE) o el Error Absoluto Medio (MAE).
    • Categ√≥ricos: Se aplican m√©tricas como la precisi√≥n o el F1-score.

    Error de Estimaci√≥n: Datos Num√©ricos¬∂

    Para datos num√©ricos, el error de estimaci√≥n se calcula como la diferencia entre el valor real y el valor predicho por el modelo. Este error se expresa matem√°ticamente como:

    $$\text{Error} = \text{Valor real} - \text{Valor predicho}$$

    El objetivo es minimizar esta diferencia. Las m√©tricas comunes para evaluar este error incluyen:

    • Error Absoluto Medio (MAE): Promedio de los valores absolutos de las diferencias.
    • Error Cuadr√°tico Medio (MSE): Promedio de los cuadrados de las diferencias, sensible a valores at√≠picos.
    • Ra√≠z del Error Cuadr√°tico Medio (RMSE): Ra√≠z cuadrada del MSE, proporcionando una m√©trica en las mismas unidades que los datos.

    Estas m√©tricas permiten evaluar la precisi√≥n del modelo y ajustar sus par√°metros para mejorar las predicciones.

    Este resultado nos muestra el error de estimaci√≥n para cada muestra (o fila). Para resumir este error, utilizamos m√©tricas de evaluaci√≥n, que se dividen en dos categor√≠as principales:

    1. M√©tricas absolutas: Las m√©tricas absolutas o no escalada miden el error sin escalar los valores. Las m√©trica absolutas m√°s ocupadas son:

      • Mean Absolute Error (MAE)

      $$\textrm{MAE}(y,\hat{y}) = \dfrac{1}{n}\sum_{t=1}^{n}\left | y_{t}-\hat{y}_{t}\right |$$

      • Root Mean Squared Error (RMSE):

      $$\textrm{RMSE}(y,\hat{y}) =(\dfrac{1}{n}\sum_{t=1}^{n}\left ( y_{t}-\hat{y}_{t}\right )^2)^{1/2}$$

    2. M√©tricas Porcentuales: Las m√©tricas porcentuales o escaladas miden el error de manera escalada, es decir, se busca acotar el error entre valores de 0 a 1, donde 0 significa que el ajuste es perfecto, mientras que 1 ser√≠a un mal ajuste. Cabe destacar que muchas veces las m√©tricas porcentuales puden tener valores mayores a 1.Las m√©trica Porcentuales m√°s ocupadas son:

      • Mean absolute percentage error (MAPE):

      $$\textrm{MAPE}(y,\hat{y}) = \dfrac{1}{n}\sum_{t=1}^{n}\left | \frac{y_{t}-\hat{y}_{t}}{y_{t}} \right |$$

    Matriz de Confusi√≥n

    La matriz de confusi√≥n es una herramienta fundamental para evaluar el rendimiento de un modelo de clasificaci√≥n. Proporciona una representaci√≥n visual que resume las predicciones realizadas por el modelo y las compara con los valores reales.

    Para la clasificaci√≥n binaria (por ejemplo, clases 0 y 1), la matriz de confusi√≥n tiene la siguiente estructura:

    Aqu√≠ se definen los componentes:

    • TP (Verdadero Positivo): El modelo predijo correctamente la clase positiva.
    • FP (Falso Positivo): El modelo predijo incorrectamente la clase positiva cuando era realmente negativa.
    • FN (Falso Negativo): El modelo predijo incorrectamente la clase negativa cuando era realmente positiva.
    • TN (Verdadero Negativo): El modelo predijo correctamente la clase negativa.

    En resumen, TP y TN representan las predicciones correctas, mientras que FP y FN indican los errores de clasificaci√≥n del modelo.

    M√©tricas de Evaluaci√≥n

    En el contexto de clasificaci√≥n, el objetivo es maximizar la cantidad de TP (Verdaderos Positivos) y TN (Verdaderos Negativos) y minimizar los FP (Falsos Positivos) y FN (Falsos Negativos). Para evaluar el rendimiento del modelo, se utilizan las siguientes m√©tricas:

    1. Accuracy (Precisi√≥n Global):

      • Mide la proporci√≥n de predicciones correctas (positivas y negativas) sobre el total de casos.

      $$accuracy(y,\hat{y}) = \dfrac{TP+TN}{TP+TN+FP+FN}$$

    2. Recall (Sensibilidad o Tasa de Verdaderos Positivos):

      • Eval√∫a la capacidad del modelo para identificar correctamente las muestras positivas. $$recall(y,\hat{y}) = \dfrac{TP}{TP+FN}$$

    3. Precision (Precisi√≥n Positiva):

      • Indica la proporci√≥n de verdaderos positivos sobre todas las predicciones positivas. $$precision(y,\hat{y}) = \dfrac{TP}{TP+FP} $$

    4. F-score (F1-Score):

      • Es la media arm√≥nica de la precisi√≥n y el recall, proporcionando un balance entre ambos. $$fscore(y,\hat{y}) = 2\times \dfrac{precision(y,\hat{y})\times recall(y,\hat{y})}{precision(y,\hat{y})+recall(y,\hat{y})} $$

    Veamos un ejemplo pr√°ctico donde calculamos estas m√©tricas utilizando sklearn.metrics:

recall = recall_score(df['y_real'], df['y_pred'])
precision = precision_score(df['y_real'], df['y_pred'])

    Curva ROC y AUC¬∂

    La curva ROC (Receiver Operating Characteristic) es una herramienta gr√°fica que eval√∫a el rendimiento de algoritmos de clasificaci√≥n binaria, representando visualmente el desempe√±o del modelo. Se construye trazando la tasa de verdaderos positivos (TPR) frente a la tasa de falsos positivos (FPR) a diferentes umbrales de decisi√≥n, permitiendo analizar el equilibrio entre sensibilidad y especificidad.

    La curva se genera calculando la TPR y la FPR para distintos umbrales, comenzando generalmente con un valor de 0.5, aunque se pueden usar valores entre 0 y 1. Esto permite observar c√≥mo cambia el rendimiento del clasificador al ajustar los umbrales.

    Definici√≥n de las tasas:

    • Tasa de verdaderos positivos (TPR) o Sensibilidad:

      $$ TPR = \dfrac{TP}{TP + FN} $$

      donde TP es el n√∫mero de Verdaderos Positivos y FN es el n√∫mero de Falsos Negativos. El TPR mide la probabilidad de que una instancia verdaderamente positiva sea correctamente clasificada como positiva.

    • Tasa de falsos positivos (FPR) o 1 - Especificidad:

      $$ FPR = \dfrac{FP}{FP + TN} $$

      donde FP es el n√∫mero de Falsos Positivos y TN es el n√∫mero de Verdaderos Negativos. La FPR mide la frecuencia con la que una instancia verdaderamente negativa se clasifica incorrectamente como positiva, es decir, una "falsa alarma".

    √Årea Bajo la Curva (AUC)

    El AUC (√Årea Bajo la Curva) resume el rendimiento del clasificador:

    • AUC = 0.5: Sin poder predictivo (predicci√≥n aleatoria).
    • AUC = 1.0: Clasificador perfecto.

    Un mayor AUC indica mejor rendimiento; AUC < 0.5 sugiere predicciones inversas.
